JNI出坑

1.创建JniUtils，定义native方法

public class JniUtils{
public native String getCString();
}

2.进行build编译操作，然后进入项目的build\intermediates\classes\debug目录下，
用javah命令对JniUtils生成.h文件
eg：

javah -jni com.eagle.hellojni.JniUtils

生成了==》com_eagle_hellojni_JniUtils.h

3.在项目下创建jni目录
将第二部生成的com_eagle_hellojni_JniUtils.h文件剪切过来，并创建一个JniUtils.c文件（命名随意）

.h文件

.c文件

4.在jni目录下建立Android.mk文件

LOCAL_PATH := $(call my-dir)

include $(CLEAR_VARS)

LOCAL_DIR = ../..
LOCAL_MODULE    := com_eagle_hellojni_JniUtils
LOCAL_SRC_FILES := JniUtils.c

include $(BUILD_SHARED_LIBRARY)

5.然后进入该jni目录运行ndk-build命令生成.so等文件

6.然后就可以通过连接上库，调用jni的方法了

  static {
    System.loadLibrary(" com_eagle_hellojni_JniUtils");
}

as的配置：

  1. gradle.properties中加入android.useDeprecatedNdk=true
  2. 配置ndk路径：local.properties文件中添加：ndk.dir=/Users/linquandong/Desktop/Android/android-ndk

  3. 根据项目结构配置jni的映射路径

    sourceSets {
main {
    jniLibs.srcDirs = ['libs'] // 配置此处才会打包jni 的.so 文件
}
}
